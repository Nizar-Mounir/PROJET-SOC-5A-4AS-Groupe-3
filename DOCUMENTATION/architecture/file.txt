L’architecture mise en place repose sur un ensemble de machines virtualisées permettant de simuler les composants essentiels d’un système d’information minimal. 
L’objectif est de créer un environnement cohérent et isolé, capable de générer des évènements variés pour la supervision et la détection.
Toutes les machines communiquent via Tailscale, utilisé comme réseau privé chiffré. 
Cette approche garantit l’isolation de l’environnement : aucun service n’est exposé sur Internet, et l’accès à l’infrastructure est strictement contrôlé.

L’environnement est constitué des machines suivantes : 
- Windows Server 2022 : VM 109 (win-server)
Serveur Windows configuré pour héberger des services permettant de générer des événements système, réseau et sécurité.
- Windows Client : VM 110 (win-client)
Poste utilisateur simulé, utilisé pour représenter un poste de travail standard au sein de l’environnement.
- Windows Domain Controller : VM 111 (win-DC)
Contrôleur de domaine Active Directory.Cette machine constitue le cœur de l’environnement Windows et permet l’analyse des événements d’authentification, de gestion de comptes et de politique de sécurité.
- Linux Debian : VM 121 (debian)
Serveur destiné à représenter un hôte Unix dans l’architecture. 
Il génère des logs liés aux commandes, aux accès fichiers, aux scans et aux actions système.
- VM SOC : 
Cette VM centralise l’ensemble des logs transmis par les autres machines via Splunk Universal Forwarder. Elle fournit également l’interface d’investigation IRIS et l’orchestration via n8n.

Chaque machine est instrumentée avec les outils adaptés à son système d’exploitation :
- Splunk Universal Forwarder pour la collecte centralisée
- Sysmon Modular sur les machines Windows
- Kunai + logs natifs sur Linux
- Audit Policy sur Windows pour renforcer la journalisation

→ L’ensemble des journaux est transmis vers la plateforme SOC pour indexation, corrélation et analyse.
